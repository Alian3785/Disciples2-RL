# Консольная визуализация GridWorld Combat Environment

Этот файл содержит консольную (текстовую) визуализацию для среды GridWorldCombatEnv с использованием только стандартной библиотеки Python.

## Особенности

### 🎯 Визуализация без внешних зависимостей
- **Только стандартная библиотека Python** - не требует pygame или других библиотек
- **Цветная консоль** (если поддерживается терминалом)
- **ASCII-арт сетка** 10x10 с четким отображением
- **Реал-тайм обновление** состояния игры

### 🎮 Интерактивное управление
- **SPACE** - Пауза/Продолжение симуляции
- **R** - Сброс симуляции
- **+/-** - Регулировка скорости (0.1x - 3.0x)
- **Q/ESC** - Выход

### 📊 Детальная статистика
- Полная информация об агенте (уровень, позиция, wounded статус)
- Состояние всех врагов (позиции, уровни, живые/мертвые)
- Подробная статистика (шаги, награды, движения, бои, лечения, смерти)
- Отслеживание истории перемещений агента

## Запуск

### Базовый запуск
```bash
python visualize_console.py
```

### Запуск с указанием чекпоинта
```bash
python visualize_console.py --checkpoint ./checkpoints2/ppo_model_800k_steps.zip
```

### Запуск с настройкой скорости
```bash
python visualize_console.py --speed 2.0
```

## Игровые элементы

### Символы на карте
- **A** - Агент (нормальное состояние)
- **W** - Агент в wounded состоянии
- **H** - Клетка лечения (0,0)
- **1** - Враг 1 (уровень 1)
- **2** - Враг 2 (уровень 2)
- **3** - Враг 3 (уровень 3)
- **.** - Пустая клетка
- **■** - Границы сетки

### Цветовая кодировка
- 🟢 **Зеленый** - Агент (нормальный)
- 🔴 **Красный** - Агент (wounded)
- 🟡 **Желтый** - Клетка лечения
- 🔴 **Красный** - Враг 1
- 🟠 **Оранжевый** - Враг 2
- 🟣 **Фиолетовый** - Враг 3

## Структура кода

### Основной класс

**ConsoleVisualizer**: Основной класс консольной визуализации
- `_load_model()` - Загрузка модели и среды
- `_draw_grid()` - Отрисовка игрового поля
- `_draw_ui()` - Отрисовка интерфейса и статистики
- `_step_simulation()` - Выполнение одного шага симуляции
- `run()` - Основной игровой цикл

### Ключевые особенности

- **Автоматическая очистка экрана** между кадрами
- **Поддержка цвета в терминале** (ANSI escape codes)
- **Fallback для систем без цвета**
- **Отслеживание зацикливаний** и автоматический сброс
- **Детальное логирование** каждого 10-го шага
- **Статистика в реальном времени**

## Требования

- **Python 3.7+**
- **stable-baselines3** (для загрузки модели)
- **gymnasium** (для окружения)

## Использование

### Пример 1: Базовая визуализация
```python
from visualize_console import ConsoleVisualizer

# Создание визуализатора
visualizer = ConsoleVisualizer("./checkpoints2/ppo_model_800k_steps.zip")

# Настройка скорости
visualizer.game_speed = 1.5

# Запуск визуализации
visualizer.run()
```

### Пример 2: Программное использование
```python
def demo_console_viz():
    visualizer = ConsoleVisualizer("./checkpoints2/ppo_model_800k_steps.zip")
    visualizer.game_speed = 2.0  # Быстрее
    visualizer.run()
```

## Диагностика и отладка

### Отладочная информация
- Каждые 10 шагов выводится детальная информация
- Отслеживается история последних 5 позиций агента
- Показывается позиция и статус всех врагов

### Защита от зацикливания
- Автоматическое обнаружение, если агент не двигается 8+ шагов
- Лимит шагов (300) для предотвращения бесконечных циклов
- Автоматический сброс окружения при обнаружении проблем

### Статистика
- **moves** - Количество перемещений агента
- **combats** - Количество боев
- **heals** - Количество лечений
- **deaths** - Количество поражений

## Советы по использованию

### Для оптимального просмотра:
1. **Увеличьте размер терминала** до 120x40 символов
2. **Используйте терминал с поддержкой цвета** (рекомендуется)
3. **Начните со скорости 1.0x**, регулируйте по вкусу
4. **Используйте паузу (SPACE)** для детального анализа

### Для отладки:
1. **Следите за историей позиций** в разделе "Отладка"
2. **Проверяйте статистику** для понимания прогресса
3. **Используйте R для сброса** при обнаружении проблем

## Преимущества консольной версии

### ✅ Плюсы:
- **Легкая** - не требует графических библиотек
- **Кросс-платформенная** - работает на любой ОС
- **Быстрая** - минимальные требования к ресурсам
- **Надежная** - только стандартные библиотеки
- **Детальная** - полная информация в текстовом формате

### ⚠️ Ограничения:
- **Нет графического интерфейса** - только текст
- **Зависит от терминала** - цвета могут не поддерживаться
- **Ограниченное интерактивное управление** - не все терминалы поддерживают

- 🎯 Наблюдения за перемещениями агента
- 👹 Отслеживания стратегии атаки врагов
- 💊 Мониторинга процесса лечения
- 📈 Анализа статистики и прогресса
