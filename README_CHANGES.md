# Changes Log Management

Этот проект использует файл `changes.md` для отслеживания значительных изменений и коммитов.

## Структура changes.md

Файл `changes.md` содержит:

- **Unreleased**: Текущие изменения, которые еще не выпущены
- **Версии**: Исторические версии с датами выпуска

### Категории изменений:

- **Added**: Новые функции и файлы
- **Changed**: Изменения в существующей функциональности
- **Fixed**: Исправления ошибок
- **Removed**: Удаленные функции

## Автоматическое обновление

### 1. Git Hook (post-commit)

После каждого коммита автоматически запускается скрипт `update_changes.py`, который добавляет информацию о коммите в раздел "Unreleased" файла `changes.md`.

### 2. Ручное обновление

Для ручного обновления выполните:

```bash
python update_changes.py --days 7
```

Это добавит все коммиты за последние 7 дней в файл `changes.md`.

### 3. Параметры скрипта

- `--days N`: Количество дней для поиска коммитов (по умолчанию 7)

## Создание нового релиза

1. Создайте новую версию в формате `[Major.Minor.Patch] - YYYY-MM-DD`
2. Перенесите содержимое из раздела "Unreleased" в новый раздел версии
3. Очистите раздел "Unreleased"

## Примеры записей

### Автоматически добавляемые коммиты:
- Добавлена новая функция обучения (abc12345)
- Исправлена ошибка в расчете награды (def67890)
- Удален устаревший код (ghi01234)

### Ручные записи:
- **Added**: Реализована новая архитектура модели
- **Changed**: Обновлена версия библиотеки stable-baselines3
- **Fixed**: Исправлена проблема с сохранением чекпоинтов
- **Removed**: Удалена поддержка устаревшего формата данных

## Полезные команды

```bash
# Показать последние коммиты
git log --oneline -10

# Показать изменения за последние 7 дней
git log --since="7 days ago" --oneline

# Создать новую версию
# 1. Обновить changes.md вручную
# 2. Закоммитить изменения
# 3. Создать git tag
git tag -a v0.2.0 -m "Version 0.2.0"
```
